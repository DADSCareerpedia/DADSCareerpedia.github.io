Queries on the database:¶

We have successfully created a database with electoral bond data we have total 4 tables and by looking the ER diagram you can see the relations between them

  • There is a relation between donordata table and bonddata table with unique_key as the common column.
  • There is a relation between reciverdata table and bonddata table with unique_key as the common column.
  • There is a relation between donordata table and bankdata table with paybranchcode and branchcode acting as the common column.
  • There is a relation between receiverdata table and bankdata table with paybranchcode and branchcode acting as the common column.

– 5. In which month most amount is spent on bonds ?

– Output gives you a month number

WITH city_bond_spending_cte AS (
    SELECT MONTH(d.PurchaseDate) AS `Month`, SUM(b.Denomination) AS 'city_bond_spending'   
    FROM donordata d
    JOIN bonddata b ON b.unique_key = d.unique_key
    GROUP BY `Month`
)
SELECT *
FROM city_bond_spending_cte
WHERE city_bond_spending = (
    SELECT MAX(city_bond_spending)
    FROM city_bond_spending_cte
);

Output:

+-------+--------------------+
| Month | city_bond_spending |
+-------+--------------------+
|     4 |        29808873000 |
+-------+--------------------+

Allow me to clarify the query I’ve written: It’s straightforward. I’ve established a Common Table Expression (CTE) containing a query that provides information about bonds spending in each month. This query is then referenced, enabling us to identify the month with the highest spending. Below is the query for monthly spending.

SELECT MONTH(d.PurchaseDate) AS `Month`, SUM(b.Denomination) AS 'city_bond_spending'   
FROM donordata d
JOIN bonddata b ON b.unique_key = d.unique_key
GROUP BY `Month`

and the output is :

Output:

+-------+--------------------+
| Month | city_bond_spending |
+-------+--------------------+
|     4 |        29808873000 |
|    10 |        28221903000 |
|     1 |        22175930000 |
|     7 |        13981937000 |
|    12 |         2321000000 |

This is the same main query the only difference is instead of month number it gives you month name.

WITH city_bond_spending_cte AS (
    SELECT MONTHNAME(d.PurchaseDate) AS `Month`, SUM(b.Denomination) AS 'city_bond_spending'  
    FROM donordata d
    JOIN bonddata b ON b.unique_key = d.unique_key
    GROUP BY `Month`
)
SELECT *
FROM city_bond_spending_cte
WHERE city_bond_spending = (
    SELECT MAX(city_bond_spending)
    FROM city_bond_spending_cte
);

Output:

+-------+--------------------+
| Month | city_bond_spending |
+-------+--------------------+
| April |        29808873000 |
+-------+--------------------+

We can write the same above query using subqueries(Derived tables):

SELECT *
FROM (
    SELECT MONTH(d.PurchaseDate) AS `Month`, SUM(b.Denomination) AS 'city_bond_spending'   
    FROM donordata d
    JOIN bonddata b ON b.unique_key = d.unique_key
    GROUP BY `Month`
) AS city_bond_spending_subquery
WHERE city_bond_spending = (
    SELECT MAX(city_bond_spending)
    FROM (
        SELECT MONTH(d.PurchaseDate) AS `Month`, SUM(b.Denomination) AS 'city_bond_spending'   
        FROM donordata d
        JOIN bonddata b ON b.unique_key = d.unique_key
        GROUP BY `Month`
    ) AS inner_subquery
);

Wow, that seems complex! In the query, we’re using the same calculation both in the FROM and WHERE clauses, which can be quite intricate. However, with Common Table Expressions (CTEs), the results are stored temporarily, making the query more efficient. This is why CTEs are often favored over subqueries and its easily readable.

– 13. Find the 2nd higest donor in terms of amount he paid?

SELECT purchaser, SUM(Denomination) as 'purchaser total donation'
FROM donordata d
JOIN bonddata b ON d.Unique_key = b.Unique_key
GROUP BY purchaser 
HAVING `purchaser total donation` = (
                                        SELECT DISTINCT(SUM(Denomination)) as 'purchaser total donation'
                                        FROM donordata d
                                        JOIN bonddata b ON d.Unique_key = b.Unique_key
                                        GROUP BY purchaser
                                        ORDER BY `purchaser total donation` DESC
                                        LIMIT 1
                                        OFFSET 1) ;

Output:

+-----------------------------------------------+--------------------------+
| purchaser                                     | purchaser total donation |
+-----------------------------------------------+--------------------------+
| MEGHA ENGINEERING AND INFRASTRUCTURES LIMITED |               9660000000 |
+-----------------------------------------------+--------------------------+

– 15. Find the party which received the second highest number of bonds?

SELECT PartyName, COUNT(r.Unique_key) as 'party received'
FROM receiverdata r
JOIN bonddata b ON r.Unique_key = b.Unique_key
GROUP BY PartyName
HAVING `party received` = (
                            SELECT DISTINCT(COUNT(r.Unique_key)) as 'party received'
                            FROM receiverdata r
                            JOIN bonddata b ON r.Unique_key = b.Unique_key
                            GROUP BY PartyName
                            ORDER BY `party received` DESC
                            LIMIT 1
                            OFFSET 1);

Output:

+------------------------------+----------------+
| PartyName                    | party received |
+------------------------------+----------------+
| ALL INDIA TRINAMOOL CONGRESS |           3305 |
